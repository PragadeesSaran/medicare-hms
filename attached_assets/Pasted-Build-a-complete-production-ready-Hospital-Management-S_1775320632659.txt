=== BACKEND API (Node.js + Express) ===

Build these REST APIs:

AUTH:
POST /api/auth/login - returns JWT token + user info
POST /api/auth/register - register new user

PATIENTS:
GET /api/patients - list all (paginated, searchable)
GET /api/patients/:id - get one
POST /api/patients - create
PUT /api/patients/:id - update
GET /api/patients/:id/history - appointment history

DOCTORS:
GET /api/doctors - list all
GET /api/doctors/:id/slots?date=YYYY-MM-DD - available slots
POST /api/doctors - create
PUT /api/doctors/:id/availability - toggle

APPOINTMENTS:
GET /api/appointments - list all
POST /api/appointments - book new
PUT /api/appointments/:id/cancel - cancel
PUT /api/appointments/:id/complete - complete
GET /api/appointments/doctor/:id - by doctor
GET /api/appointments/today - today's count

PRESCRIPTIONS:
GET /api/prescriptions - list
POST /api/prescriptions - create
GET /api/prescriptions/patient/:id - by patient

BILLING:
GET /api/bills - list all
POST /api/bills/generate - generate bill
PUT /api/bills/:id/pay - mark paid
GET /api/bills/stats - total revenue, pending

DASHBOARD:
GET /api/dashboard/stats - counts for all modules
GET /api/dashboard/chart-data - last 7 days appointment data

=== DATABASE TABLES (PostgreSQL) ===

users: id, username, password(hashed), email, role, is_active, created_at
departments: id, name
patients: id, user_id, full_name, dob, gender, blood_group, phone, address
doctors: id, user_id, dept_id, full_name, specialization, experience_yrs, is_available
appointments: id, patient_id, doctor_id, appointment_date, time_slot, status, notes, created_at
prescriptions: id, appointment_id, doctor_id, diagnosis, medicines, dosage_instructions, issued_date
bills: id, patient_id, appointment_id, consultation_fee, medicine_cost, total_amount, payment_status, bill_date
